Definition at line 261 of file UI.cpp.

void UI::keyboardStateProcessing ( )
private

The method processes the keyboard state if the yaw-pitch-roll camera and the object firing control.

Remarks

If no modifier keys are pressed:

  • Left: Increases the yaw.
  • Right: Decreases the yaw.
  • Up: Increases the pitch.
  • Down: Decreases the pitch.

If the control or Z key is down:

  • Left: Increases the roll.
  • Right: Decreases the roll.
  • Up: Moves the camera forward.
  • Down: Moves the camera backward.

If the shift or S key is down:

  • Left: Moves the camera left.
  • Right: Moves the camera right.
  • Up: Moves the camera up.
  • Down: Moves the camera down.

These options are limited to 5 per second.

  • 1: Drops a cube.
  • 2: Drops a sphere.
  • 3: Drops a teapot.
  • 4: Drops a torus.
  • 5: Fires a box from the camera.
  • 6: Fires a sphere from the camera.
  • 7: Fires a sphere from a random point on a circle around the scene.
  • 8: Fires a box from a random point on a circle around the scene.
  • 9: Fires a teapot from a random point on a circle around the scene.
  • 0: Fires a donut (torus) from a random point on a circle around the scene.

If the F is down: These options are not time limited.

  • 1: Drops a cube.
  • 2: Drops a sphere.
  • 3: Drops a teapot.
  • 4: Drops a torus.
  • 5: Fires a box from the camera.
  • 6: Fires a sphere from the camera.
  • 7: Fires a sphere from a random point on a circle around the scene.
  • 8: Fires a box from a random point on a circle around the scene.
  • 9: Fires a teapot from a random point on a circle around the scene.
  • 0: Fires a donut (torus) from a random point on a circle around the scene.

void UI::processEvents ( )

The method handles the SFML event processing and calls the keyboard state processor method.

This method processes all events in the current SFML event queue and calls the corresponding processing method. At the end it calls the keyboard state processor method, outside the event loop.
